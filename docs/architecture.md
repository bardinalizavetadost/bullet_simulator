# Архитектура системы

## Общее описание
Система представляет собой симулятор баллистической траектории с учетом сопротивления воздуха. Архитектура построена по принципу разделения ответственности между компонентами.

## Основные компоненты

### 1. Физический движок (Physics)
Отвечает за расчет движения пули под действием сил. Основные функции:
- Расчет ускорения на основе действующих сил
- Обновление положения и скорости пули
- Учет гравитации и сопротивления воздуха
- Отслеживание метрик полета (высота, дальность, путь)

### 2. Модель пули (Bird)
Представляет собой игровой объект с физическими свойствами. Включает:
- Управление состоянием полета (запущена/столкнулась)
- Хранение истории движения
- Визуализацию пули и действующих на нее сил
- Взаимодействие с физическим движком

### 3. Графический интерфейс (MyPanel)
Основной компонент пользовательского интерфейса. Отвечает за:
- Отображение симуляции
- Управление пользовательскими элементами (кнопки, слайдеры)
- Обработку ввода пользователя
- Автоматическую прокрутку для отслеживания пули
- Отображение информации о текущем состоянии

### 4. Конфигурация (PhyConfig)
Хранит параметры физической модели:
- Характеристики пули (масса, калибр, баллистический коэффициент)
- Параметры запуска (угол, начальная энергия)
- Атмосферные условия (температура, давление)
- Предустановленные конфигурации боеприпасов

## Принципы работы
1. Пользователь настраивает параметры через интерфейс
2. При запуске создается экземпляр Bird с заданной конфигурацией
3. Физический движок обновляет состояние пули на каждом шаге симуляции
4. Графический интерфейс отображает текущее состояние и историю полета
5. Пользователь может управлять симуляцией (пауза, откат времени)
